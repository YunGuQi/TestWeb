# 小红书帖子物料全自动制作 SOP（以《性格城市匹配》为例）

本文档总结了小红书引流帖从“需求脑暴”到“一键出图”的完整工作流与底层设计逻辑，作为后续开发其他测试帖子物料的标准参考（SOP）。

---

## 1. 使用的技能池与唤醒词 (Skills & Triggers)

在启动帖子物料设计前，必须先理清“怎么发才能火”的底层逻辑，并产出高质量的网感文案。我们使用了以下内置技能：

### A. 策略脑暴阶段
- **唤醒词组合**：`/best-minds /brainstorming /grill-me`
- **结合上下文**：`@[CLAUDE.md] 按照找的对标账户和文案分析，该如何设计帖子，请分析`
- **核心作用**：调用顶级运营专家的视角发散思维，严格拷问方案的落地可行性，确定“痛点引入 -> 极简测试 -> 评论互动 -> 主页引流”的漏斗模型。

### B. 文案裂变与包装阶段（文案生产三步法）
- **唤醒词组合**：`/xhs-content-creator`
- **核心作用与规范**：绝对不能只生硬模仿对标账号，必须按照以下三步法生成文案：
  1. **提取魂**：提炼对标账号打的痛点情绪和底层逻辑。
  2. **造骨肉 (`/xhs-content-creator`)**：利用爆款钩子公式（制造好奇反差、故事代入、逆向反常识等），将 1 个核心痛点裂变为 5 种不同方向的开头和文案框架。
  3. **定妆 (`/xhs-content-creator`)**：将 AI 生成的框架直接输出为纯正的“小红书味”，加入“家人们/姐妹们”、精准控制 Emoji、打碎成长段落，完美避开同质化查重。

## 2. 帖子设计与排版逻辑 (Design & Layout)

小红书图文帖子必须在 1 秒内抓住眼球，因此设计上遵循以下规范：

1. **绝对尺寸限制**：所有生成的图片必须严格锁定 **1080 × 1440 (3:4)** 比例，防止小红书自动裁剪导致信息丢失。
2. **视觉语言统一**：
   - **底色调**：根据具体测试主题设定（如牛皮纸/工业纸张质感 `var(--bg-kraft)`），配合轻微噪点增加高级感。
   - **排版风格**：**根据具体的测试来设计风格**（例如《性格城市》采用复古票据风、工业风。大字号黑体压阵，红笔手绘感高亮核心信息）。
   - **组件复用**：直接将主网站的核心视觉锚点（如“心灵逃跑车票”）1:1 像素级复刻到帖子末尾作为钩子。
3. **文案呈现结构 (6+1 模式)**：
   - **首图**：**根据具体的测试来设计痛点**进行情绪暴击（例如“现在的城市真的懂你吗？”），并辅以明确的左滑提示。
   - **内容图 (P2-P6)**：5 道精简版核心测试题，排版清晰，选项分明，左上角标明进度。
   - **尾图 (P7)**：测试结果样例钩子，明确下达互动指令（例如“评论区留下选项字母 @问一问，获取简版诊断”）。

## 3. 自动化出图工作流 (Automated Workflow)

为了彻底消除设计软件（如 PS/Figma）带来的重复劳动，并严格遵循防乱码的安全红线，我们摒弃了 Python 后台脚本，直接采用 **原生前端技术闭环** 来实现一键批量出图。

### 核心工具：`post_generator.html`
这是一个独立的静态网页生成器，专门为产出帖子物料而生。

- **DOM 渲染图层**：用纯粹的 HTML + CSS（结合 TailwindCSS）排版出 7 张帖子画板，支持无限次修改文案和样式，实时预览。
- **html2canvas 引擎**：点击按钮后，前端利用 JS 将复杂的 HTML 结构（含字体、阴影、SVG 噪点）瞬间转化为纯粹的 Canvas 图像。
- **File System Access API (黑科技)**：
  - **痛点**：传统网页下载（`<a>` 标签）只能被动存入浏览器的默认“下载”文件夹，容易导致文件混乱且丢失后缀。
  - **解决方案**：引入 `window.showDirectoryPicker`。点击下载后，浏览器原生弹窗让用户**直接指定**项目物料文件夹（如 `帖子物料\性格城市匹配`）。
  - **结果**：7 张极清 PNG 图片被全自动、精确无误地批量写入指定文件夹。既保证了纯正的前端实现，又达成了极客级别的自动化存储。

## 4. 落地执行规范 (Checklist)

以后每上线一个新测试，必须在 `帖子物料/` 目录下建立同名文件夹，并标配以下文件：
1. **`物料使用说明.md`**：记录如何使用生成器和注意事项。
2. **`发帖文案与配置.md`**：存放帖子的标题、正文、Tag、以及冷启动小号预埋的剧本。
   - **【文案批量输出规范】**：在要求 AI 撰写文案时，**必须添加文案模仿输出指令，要求一次性生成 5 条不同角度的文案组合（每条均需包括完整的标题和正文内容）**。必须严格执行“文案生产三步法”（配合调用 `/xhs-social-content` 和 `/xhs-xiaohongshu-converter` 技能），既保留对标账号的爆款基因，又通过小红书平台特化避开查重。**⚠️重要提示 1：小红书标题最多只能输入 20 字（包含标点和表情符号），超过会被自动截断，因此所有发帖文案的标题字数必须严格限制在 20 字以内！** **⚠️重要提示 2：所有发帖文案的正文末尾，必须显式加入高阶互动引导话术（如：“评论区留下选项字母，并 @问一问 ，获取专属深度报告”），确保触发官方 AI 助手引流闭环。** **⚠️重要提示 3：在生成多角度文案时，必须同步修改 `post_generator.html` 中的首图代码，将 5 条文案的标题分别设计排版（合理使用 `<br>` 断句和 `<span class="highlight-red">` 高亮）到独立的 `<div id="capture-cover-X">` 中，并配置 JS 脚本一键下载这 5 张不同排版的主图。以确保每篇帖子都有视觉独立的封面，彻底规避平台的视觉查重！** 这为后续矩阵分发和 A/B 赛马测试备足了合格的弹药。
3. **`post_generator.html`**：存放对应的 HTML 自动化生成器代码。
6. **生成的 PNG 矩阵**：如 `0-首图.png`、`1-题1.png` 等。**必须存放在以“生成日期”命名的子文件夹内（例如 `2026-06-21/`）**，方便以后生成不同版本的内容并避免文件相互覆盖。生成后直接拖入手机即可发帖。

## 5. 千帆商品物料生成规范 (Qianfan Assets)

在上线新测试时，除了小红书引流图文，还需要在后台上架用于“自动发货”的虚拟商品。为了满足千帆后台的严苛限制（主图大小严格 < 5MB）且提高转化率，我们制定了以下 SOP：

1. **统一物料存放**：在对应的 `帖子物料/<测试名称>/` 下新建 `千帆/` 文件夹。
2. **极简首图生成 (Python + Playwright)**：
   - **尺寸与排版**：1:1 正方形（800x800），极简“大字报”风格。浅色背景，核心文案直击痛点，并带有类似“千帆严选”或“拍下秒发”的印章/标签。
   - **文件限制**：必须通过代码（如 `html2canvas` 或 Playwright 截图）压制生成高压缩比的 `.jpg`，保证绝对小于 5MB。
3. **商品详情文案 (`千帆商品详情文案.md`)**：
   - 结构必须精简：`[痛点拷问引言] + [能获得什么报告指标] + [明确的下单自动发货指引]`。
4. **测试流程图**：需复用 `common/` 目录下设计的《测试流程 & 购买须知》说明图（`千帆购买流程.jpg`）作为商品详情轮播图第二张。

> **⚠️ 提示：在新建任何测试的物料时，必须收到用户的明确通知/指令后，再自动执行上述千帆物料的生成脚本与文案写作。**
