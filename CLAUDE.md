# 小红书虚拟测试运营 · 核心开发与交互规范

## 1. 核心工作流统领
当前项目已全面升级为 Agent Skill 驱动模式。任何关于**爆款挖掘、H5测试开发、帖子物料生成、防风控发布**的端到端运营任务，**不再需要依赖零散的指令，必须直接唤醒专属的统筹中枢技能：`/xhs-auto-operator`**。
相关的业务拆解与 SOP 流水线规范已安全沉淀在 `.agent/skills/xhs-auto-operator/references/` 中，请严格遵从技能调度，不要自行发散。

## 2. 测试页面开发规范 (H5)
- **直接修改 HTML 文件**：修改测试页面代码时，**必须直接使用编辑器工具修改 HTML/CSS/JS 代码**，**绝对禁止**编写 Python 脚本（如 `rewrite.py`、`patch.py`）来间接修改 HTML 文件。
- **单文件原则**：每个测试产品必须保持 HTML + CSS + JS 全部内联在一个 `.html` 文件中，绝不能做成多文件分离的形式，以确保能一键上传云端部署和跨平台分享。

## 3. 交互与代码操作规范
- **称呼规范**：在与用户对话回复时，必须称呼用户为“大哥”。
- **禁止自行 Git 回滚**：不要自己用 git 回滚代码（如使用 `git checkout` 等命令），以避免已修改或未提交的内容被意外覆盖。
- **字符编码规范**：在修改或处理文件内容时，务必注意 UTF-8 的编码问题，严禁再次出现因编码设置不当导致的中文乱码。
**特别警告：绝对禁止在没有显式指定 `-Encoding utf8` 的情况下，使用 Windows PowerShell（如 `Get-Content` 或 `Set-Content`）直接处理包含中文的代码文件，这会将其错误编码并直接导致乱码毁坏文件！优先使用内部的原生文件修改工具（如 `replace_file_content`）。**
