# CLAUDE.md - 小红书虚拟测试电商项目指引

## 项目概述
本项目旨在通过小红书平台运营心理/人格测试类虚拟产品。核心流程包括：制作测试网站（秒哒/腾讯云托管） -> 产出小红书笔记文案 -> 引导用户至小红书店铺下单 -> 自动发货（阿奇索）。

## 技术栈与规范
### 前端开发
- **框架**：原生 HTML5 / CSS3 / JavaScript (ES6+)，无第三方重型框架依赖。
- **设计风格**：疗愈系（Healing style），使用浅色渐变、大圆角、卡片式布局。
- **响应式**：移动优先（Mobile First），最大宽度限制为 480px，桌面端居中显示。
- **交互规范**：
  - 点击区域（Touch Targets）不小于 48px。
  - 页面切换使用 `opacity` + `transform` 动画。
  - 进度条必须实时反馈答题进度。
- **代码注释**：所有源代码注释必须使用**中文**。

### 后端/脚本 (如有)
- **环境管理**：必须使用 `uv` 进行依赖管理和脚本执行。
- **命令示例**：
  - 安装依赖：`uv add <package>`
  - 运行脚本：`uv run python <script>.py`

## 运营准则 (红线)
1. **文案合规**：
   - 严禁出现「购买」、「付费」、「点击链接」等营销敏感词。
   - 严禁声称测试具有「医疗诊断」或「临床治疗」效果。
   - 必须在结果页底部包含「免责声明」。
2. **发布策略**：
   - 每日目标发布 5+ 条笔记。
   - 必须使用手机流量发布，严禁同 WiFi 多号操作。
